# Backlog Grooming Template

Use this template to facilitate a backlog refinement (grooming) session. The goal is to ensure upcoming stories are well-defined, estimated, and ready for sprint planning.

## Stories to Review

For each story, walk through the checklist below.

### Story: #<issue number> — <Title>

**Current state:** Draft / Needs refinement / Ready

#### Description Review

- [ ] Problem statement is clear
- [ ] User story follows "As a / I want / So that" format
- [ ] Scope is well-defined (not too large, not too vague)

#### Acceptance Criteria Review

- [ ] Acceptance criteria exist and use Given/When/Then format
- [ ] Happy path is covered
- [ ] Edge cases and error states are addressed
- [ ] Non-functional requirements noted (performance, accessibility)

#### Estimation

- **Estimate:** <story points or T-shirt size>
- **Confidence:** High / Medium / Low
- **Notes:** <any estimation concerns>

#### Dependencies

- Blocked by: <issue number or "none">
- Blocks: <issue number or "none">

#### Decision

- [ ] **Ready** — meets Definition of Ready, can be planned
- [ ] **Needs work** — requires changes before planning (see action items)
- [ ] **Split** — too large, break into smaller stories
- [ ] **Deferred** — not a priority for upcoming sprints

---

*Copy the story section above for each story reviewed.*

## Definition of Ready

A story is ready for sprint planning when:

- [ ] User story and acceptance criteria are complete
- [ ] Estimated by the team
- [ ] Dependencies identified and unblocked (or plan to unblock)
- [ ] No open questions remaining
- [ ] Small enough to complete in one sprint
